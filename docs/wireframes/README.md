## Goal

**Reuse blocks** so every screen shares the same geometry, colors, and spacing ‚Äî not copy-paste in `generate-screens.mjs`.

## What lives in this folder

| File | Role |
|------|------|
| `components.mjs` | Primitives: colors, shapes, `fieldLabel` / `fieldLabelBlock`, `component()` / `componentContent()`, `appShell` |
| `blocks.mjs` | **Auth & form blocks** (SSO row, fields, terms, `authCard`, register-org painters) ‚Äî **edit here first** for auth UI |
| `generate-template.mjs` | Kit section builders **S01‚ÄìS37** (table, modal, workflow canvas, ‚Ä¶) ‚Äî import-only, no CLI output |
| `generate-screens.mjs` | Screen generators ‚Üí `docs/use-cases/{domain}/{use-case}/*.excalidraw` |
| `assets/axis-logo.svg` | **Canonical brand mark** (three nodes, no wordmark) |
| `logo.mjs` | `buildAxisLogo()` ‚Äî embeds SVG in Excalidraw via `files` |
| `app-shell.excalidraw` + `.svg` | Shared app chrome (only committed artifact besides source `.mjs` here) |

There is **no** `_template.excalidraw` catalog (removed). Kit geometry lives in `.mjs` only; screen outputs live under each use-case folder.

## Layers (single source per layer)

| Layer | File | Screens use |
|-------|------|-------------|
| Primitives | `components.mjs` | `rect`, `text`, `fieldLabelBlock`, `component()` |
| Blocks | `blocks.mjs` | `authFormField`, `placeAuthExternalSignIn`, `paintRegisterOrg*`, ‚Ä¶ |
| Kit sections | `generate-template.mjs` | `component(buildTable, cx, cy)` etc. |

## Workflow

```bash
# 1. Edit source (see Agent contract ‚Äî which file to touch)
#    components.mjs   ‚Äî label/help spacing, REQUIRED_MARKER_GAP, colors
#    blocks.mjs       ‚Äî auth fields, SSO, register-org painters, AUTH_FIELD_STACK_GAP
#    generate-template.mjs ‚Äî only when adding/changing S01‚ÄìS37 builders

# 2. Regenerate screens (twice; second run must produce empty git diff)
node docs/wireframes/generate-screens.mjs
node docs/wireframes/generate-screens.mjs

# Optional: one use-case only (substring match on screen key)
SCREEN_FILTER=register-org node docs/wireframes/generate-screens.mjs
SCREEN_FILTER=email-confirmation node docs/wireframes/generate-screens.mjs

# 3. SVG previews (Kroki)
docs/scripts/generate-wireframes.ps1 -Filter register-org
```

`app-shell` is generated by `generate-screens.mjs` like any other screen.

## Agent contract

Single source for spacing and composition rules. **Change constants here, not per-screen magic numbers.**

### Where to edit what

| Change | Edit | Regenerate |
|--------|------|------------|
| Label ‚Üí help ‚Üí input spacing, `?` icon size/colors | `components.mjs` (`fieldLabelBlock`) | Screens using `authFormField` / `fieldLabelBlock` |
| Gap between stacked auth fields, card width, SSO row | `blocks.mjs` | Auth screens + register-org |
| New register-org field list / help copy | `blocks.mjs` (`REGISTER_ORG_*_FIELDS`, painters) | `SCREEN_FILTER=register-org` |
| Table, modal, workflow canvas | `generate-template.mjs` | Screens that `component(buildXxx)` |
| Screen layout only (positions, copy) | `generate-screens.mjs` | That screen or filter |

### Spacing contract (shared constants)

All auth fields that use `authFormField`, `authReadOnlyValueField`, or `authSlugPreviewField` share these values automatically via `fieldLabelBlock` and `AUTH_FIELD_STACK_GAP`.

| Constant | Value | Defined in | Meaning |
|----------|------|------------|---------|
| `REQUIRED_MARKER_GAP` | 10px | `components.mjs` | Label text ‚Üí red `*` |
| `FIELD_LABEL_H` | 16px | `components.mjs` | Label row height |
| `FIELD_LABEL_TO_HELP_GAP` | 6px | `components.mjs` | Label row ‚Üí help text row |
| `FIELD_HELP_TEXT_H` | 14px | `components.mjs` | Help text line height |
| `FIELD_HELP_TO_INPUT_GAP` | 8px | `components.mjs` | Help row (or label if no help) ‚Üí input top |
| `HELP_ICON_SIZE` | 12px | `components.mjs` | `?` circle on help row |
| `HELP_TEXT_ICON_GAP` | 6px | `components.mjs` | `?` icon ‚Üí help copy (same row) |
| `AUTH_FIELD_STACK_GAP` | 12px | `blocks.mjs` | Bottom of one field block ‚Üí next field |
| `AUTH_CARD_FOOTER_ZONE` | 44px | `blocks.mjs` | Footer divider + link inside card |
| `measureAuthCardHeight` | ‚Äî | `blocks.mjs` | Card height from layout cursor **and** max painted element bottom |
| `authScreenCanvasHeight` | ‚Äî | `blocks.mjs` | Gray background `H` when card exceeds 700px |
| `AUTH_CARD_W` | 440px | `blocks.mjs` | Centered auth card width |
| `AUTH_CARD_PAD_X` | 24px | `blocks.mjs` | Card inner horizontal padding |

**Input control height:** 40px (inside auth field builders). **Do not** use `y + 18` for auth fields ‚Äî use `authFormField` so `inputY` comes from `fieldLabelBlock`.

**Legacy screens** (settings-org, form-editor, some modals) may still use `fieldLabel()` + manual `y + 18`. They are **not** on this contract until migrated. New auth/register work **must** use blocks.

### Help text pattern (canonical)

```
Organization name *
(?) 2‚Äì100 characters. Shown on invoices‚Ä¶
[‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ input 40px ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ]
```

- `?` sits on the **help row**, **before** the help sentence (not on the label row).
- Colors: circle `C.primaryDark` stroke on `C.infoBg`; `?` and stroke use primary; help copy `C.gray700` 10px.
- Pass copy via `helpText` on `authFormField(‚Ä¶)` (or field defs in `REGISTER_ORG_ENTRY_FIELDS`).

### Which API to use (decision tree)

```
Auth / registration form?
‚îú‚îÄ Yes ‚Üí Simple card (login, forgot password)?
‚îÇ        ‚îî‚îÄ authCard() in blocks.mjs
‚îú‚îÄ Yes ‚Üí Composite (SSO + many fields, states panels)?
‚îÇ        ‚îî‚îÄ buildAuthCardHeader/Footer/SubmitButton
‚îÇ           + placeAuthExternalSignIn + authFormField / painters
‚îú‚îÄ Yes ‚Üí register-org entry?
‚îÇ        ‚îî‚îÄ paintRegisterOrgEntryFields
‚îÇ           + REGISTER_ORG_ENTRY_FIELDS (extend field defs, do not duplicate geometry)
‚îî‚îÄ No ‚Üí App shell page with table/canvas?
         ‚îî‚îÄ appShell() + component(buildTable|buildWorkflowCanvas|‚Ä¶)
```

### MUST

- Import spacing from `components.mjs` / `blocks.mjs`; **never** hardcode `6`, `8`, `12`, `18`, `440` for auth field layout.
- Use `componentContent()` or `placeAuthExternalSignIn()` for headerless SSO block ‚Äî **never** `component()` on blocks without `sectionHeader`.
- Call every generator through `runScreen(screenKey, genXxx)` (deterministic seed per screen).
- Run `generate-screens.mjs` **twice**; second run `git diff` must be empty.
- Regenerate `.svg` for touched `.excalidraw` (Kroki / `generate-wireframes.ps1`).
- Commit generator `.mjs` and regenerated artifacts together.

### MUST NOT

- Hand-edit `docs/use-cases/**/**.excalidraw` (except rare review tweaks ‚Äî prefer fixing generators).
- Recreate `_template.excalidraw` or document `node generate-template.mjs` as a catalog CLI (`generate-template.mjs` is import-only).
- Duplicate SSO buttons, auth field geometry, or help rows in `generate-screens.mjs`.
- Use `component()` on `buildAuthExternalSignInBlock` / `placeAuthExternalSignIn` output (strips wrong elements).
- Add auth help text by drawing extra `text` elements on the screen generator ‚Äî use `helpText` param.

- Use `buildAxisLogo(prefix, slotX, y, slotW)` from `logo.mjs` (re-exported in `blocks.mjs`) ‚Äî **mark + "Axis" wordmark**, not mark-only or `‚¨°  Axis` placeholder text.
- Source file: [`assets/axis-logo.svg`](assets/axis-logo.svg) ‚Äî three nodes (amber / green / blue) with a soft gradient bond (no hard stroke lines). Sync [`frontend/src/assets/axis-logo.svg`](../../frontend/src/assets/axis-logo.svg) when the mark changes.
- Wired in: `buildAuthCardHeader`, `buildAuthCardBrandBar`, `authCard`.
- `writeExcalidraw(..., elements, files)` must receive merged `files` from any screen that embeds the logo (`mergeExcalidrawFiles` in `generate-screens.mjs`).
- Open `.excalidraw` in Excalidraw to see the SVG. Kroki SVG preview may not render embedded images ‚Äî check the source file in app if preview looks empty.

## Rules (short)

1. **New auth / register UI** ‚Üí `blocks.mjs` (not `generate-template.mjs`).
2. **New large pattern** (data grid, builder layout) ‚Üí `buildXxx(y0)` in `generate-template.mjs` + `component()` on screens.
3. **`component()`** ‚Äî builders with `sectionHeader` only.
4. **`componentContent()`** ‚Äî headerless blocks from `blocks.mjs`.
5. **Spacing** ‚Äî only via constants in the Agent contract table.
