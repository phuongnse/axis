**Agents:** before changing wireframes, read **[Agent contract](#agent-contract)** below and [wireframes.md § Agent checklist](../playbooks/wireframes.md#agent-checklist). Multi-screen use cases: [wireframes.md § Multi-screen journey pattern](../playbooks/wireframes.md#multi-screen-journey-pattern).

## Goal

**Reuse blocks** so every screen shares the same geometry, flow, hierarchy, and spacing — not copy-paste in `generate-screens.mjs`.

Wireframes are low-fidelity skeletons. They document layout, user flow, content intent, form/help/error states, and action hierarchy. They do **not** document product theme, dark/light styling, decorative backdrops, final colors, shadows, or exact visual polish. Those belong to frontend components, design tokens, screenshots, and visual QA.

## What lives in this folder

| File | Role |
|------|------|
| `components.mjs` | Primitives: colors, shapes, `fieldLabel` / `fieldLabelBlock`, `component()` / `componentContent()`, `appShell` |
| `blocks.mjs` | **Auth & form blocks** (page frame, auth card skeleton, optional provider row, fields, terms, register-org painters) — **edit here first** for auth UI |
| `generate-template.mjs` | Kit section builders **S01–S37** (table, modal, workflow canvas, …) — import-only, no CLI output |
| `generate-screens.mjs` | Screen generators → `docs/use-cases/{domain}/{use-case}/*.excalidraw` |
| `assets/axis-logo.svg` | **Canonical brand mark** (three nodes, no wordmark) |
| `logo.mjs` | `buildAxisMark()` / `buildAxisLogo()` — embeds the canonical SVG in Excalidraw via `files` |
| `app-shell.excalidraw` + `.svg` | Shared app chrome (only committed artifact besides source `.mjs` here) |

There is **no** `_template.excalidraw` catalog (removed). Kit geometry lives in `.mjs` only; screen outputs live under each use-case folder.

## Layers (single source per layer)

| Layer | File | Screens use |
|-------|------|-------------|
| Primitives | `components.mjs` | `rect`, `text`, `fieldLabelBlock`, `component()` |
| Blocks | `blocks.mjs` | `buildAuthPageFrame`, `authCard`, `authFormField`, `paintRegisterOrg*`, … |
| Kit sections | `generate-template.mjs` | `component(buildTable, cx, cy)` etc. |

## Workflow

```bash
# 1. Edit source (see Agent contract — which file to touch)
#    components.mjs   — label/help spacing, REQUIRED_MARKER_GAP, colors
#    blocks.mjs       — auth page frame/card skeleton, auth fields, register-org painters, AUTH_FIELD_STACK_GAP
#    generate-template.mjs — only when adding/changing S01–S37 builders

# 2. Regenerate screens (twice; second run must produce empty git diff)
node docs/wireframes/generate-screens.mjs
node docs/wireframes/generate-screens.mjs

# Optional: one use-case only (substring match on screen key)
SCREEN_FILTER=register-org node docs/wireframes/generate-screens.mjs
SCREEN_FILTER=email-confirmation node docs/wireframes/generate-screens.mjs

# 3. SVG previews (Kroki)
docs/scripts/generate-wireframes.ps1 -Filter register-org
```

`app-shell` is generated by `generate-screens.mjs` like any other screen.

## Agent contract

Single source for spacing and composition rules. **Change constants here, not per-screen magic numbers.**

### Where to edit what

| Change | Edit | Regenerate |
|--------|------|------------|
| Label → help → input spacing, `?` icon size/colors | `components.mjs` (`fieldLabelBlock`) | Screens using `authFormField` / `fieldLabelBlock` |
| Gap between stacked auth fields, card width, auth page frame/card skeleton, optional provider row | `blocks.mjs` | Auth screens + register-org |
| New register-org field list / help copy | `blocks.mjs` (`REGISTER_ORG_*_FIELDS`, painters) | `SCREEN_FILTER=register-org` |
| Table, modal, workflow canvas | `generate-template.mjs` | Screens that `component(buildXxx)` |
| Screen layout only (positions, copy) | `generate-screens.mjs` | That screen or filter |

### Spacing contract (shared constants)

All auth fields that use `authFormField`, `authReadOnlyValueField`, or `authSlugPreviewField` share these values automatically via `fieldLabelBlock` and `AUTH_FIELD_STACK_GAP`.

### Fidelity contract

- Use grayscale structure by default.
- Use semantic colors only when they explain state: error, warning, success, disabled.
- Show decorative/background systems as a light page frame or placeholder only when they affect layout.
- Do not encode real product palette, dark/light mode, final radius, shadow, texture, or decorative motif in wireframes.
- If a visual decision needs enforcement, put it in frontend component rules/tests, not wireframe screenshots.

| Constant | Value | Defined in | Meaning |
|----------|------|------------|---------|
| `REQUIRED_MARKER_GAP` | 10px | `components.mjs` | Label text → red `*` |
| `FIELD_LABEL_H` | 16px | `components.mjs` | Label row height |
| `FIELD_LABEL_TO_HELP_GAP` | 6px | `components.mjs` | Label row → help text row |
| `FIELD_HELP_TEXT_H` | 14px | `components.mjs` | Help text line height |
| `FIELD_HELP_TO_INPUT_GAP` | 8px | `components.mjs` | Help row (or label if no help) → input top |
| `HELP_ICON_SIZE` | 12px | `components.mjs` | `?` circle on help row |
| `HELP_TEXT_ICON_GAP` | 6px | `components.mjs` | `?` icon → help copy (same row) |
| `AUTH_FIELD_STACK_GAP` | 12px | `blocks.mjs` | Bottom of one field block → next field |
| `AUTH_CARD_FOOTER_ZONE` | 44px | `blocks.mjs` | Footer divider + link inside card |
| `measureAuthCardHeight` | — | `blocks.mjs` | Card height from layout cursor **and** max painted element bottom |
| `authScreenCanvasHeight` | — | `blocks.mjs` | Auth canvas height when card exceeds default screen height |
| `AUTH_CARD_W` | 440px | `blocks.mjs` | Centered auth card width |
| `AUTH_CARD_PAD_X` | 24px | `blocks.mjs` | Card inner horizontal padding |

**Input control height:** 40px (inside auth field builders). **Do not** use `y + 18` for auth fields — use `authFormField` so `inputY` comes from `fieldLabelBlock`.

**Legacy screens** (settings-org, form-editor, some modals) may still use `fieldLabel()` + manual `y + 18`. They are **not** on this contract until migrated. New auth/register work **must** use blocks.

### Help text pattern (canonical)

```
Organization name *
(?) 2–100 characters. Shown on invoices…
[──────── input 40px ────────]
```

- `?` sits on the **help row**, **before** the help sentence (not on the label row).
- Colors: circle `C.primaryDark` stroke on `C.infoBg`; `?` and stroke use primary; help copy `C.gray700` 10px.
- Pass copy via `helpText` on `authFormField(…)` (or field defs in `REGISTER_ORG_ENTRY_FIELDS`).

### Which API to use (decision tree)

```
Auth / registration form?
├─ Yes → Simple card (login, forgot password)?
│        └─ authCard() in blocks.mjs
├─ Yes → Composite (many fields, states panels, provider entry points when implemented)?
│        └─ buildAuthCardHeader/Footer/SubmitButton
│           + authFormField / painters
├─ Yes → register-org entry?
│        └─ paintRegisterOrgEntryFields
│           + REGISTER_ORG_ENTRY_FIELDS (extend field defs, do not duplicate geometry)
└─ No → App shell page with table/canvas?
         └─ appShell() + component(buildTable|buildWorkflowCanvas|…)
```

### MUST

- Import spacing from `components.mjs` / `blocks.mjs`; **never** hardcode `6`, `8`, `12`, `18`, `440` for auth field layout.
- Use shared block builders for provider entry points when that UI is actually implemented — **never** hand-draw provider buttons in screen generators.
- Call every generator through `runScreen(screenKey, genXxx)` (deterministic seed per screen).
- Run `generate-screens.mjs` **twice**; second run `git diff` must be empty.
- Regenerate `.svg` for touched `.excalidraw` (Kroki / `generate-wireframes.ps1`).
- Commit generator `.mjs` and regenerated artifacts together.

### MUST NOT

- Hand-edit `docs/use-cases/**/**.excalidraw` (except rare review tweaks — prefer fixing generators).
- Recreate `_template.excalidraw` or document `node generate-template.mjs` as a catalog CLI (`generate-template.mjs` is import-only).
- Duplicate provider buttons, auth field geometry, or help rows in `generate-screens.mjs`.
- Show external-provider entry points in a wireframe before the provider flow is implemented and documented for that use case.
- Add auth help text by drawing extra `text` elements on the screen generator — use `helpText` param.

## Brand mark (`buildAxisMark` / `buildAxisLogo`)

- Use `buildAxisMark(prefix, x, y, size)` for current public auth headers. Use `buildAxisLogo(prefix, slotX, y, slotW)` only when a full mark + wordmark lockup is explicitly needed. Never use `⬡  Axis` placeholder text.
- Source file: [`assets/axis-logo.svg`](assets/axis-logo.svg) — three nodes (amber / green / blue) with a soft gradient bond (no hard stroke lines). Sync [`frontend/src/assets/axis-logo.svg`](../../frontend/src/assets/axis-logo.svg) when the mark changes.
- Wired in: `buildAuthBrandHeader`, `buildAuthCardHeader`, `buildAuthCardBrandBar`, `authCard`.
- `writeExcalidraw(..., elements, files)` must receive merged `files` from any screen that embeds the logo (`mergeExcalidrawFiles` in `generate-screens.mjs`).
- Open `.excalidraw` in Excalidraw to see the SVG. Kroki SVG preview may not render embedded images — check the source file in app if preview looks empty.

## Rules (short)

1. **New auth / register UI** → `blocks.mjs` (not `generate-template.mjs`).
2. **New large pattern** (data grid, builder layout) → `buildXxx(y0)` in `generate-template.mjs` + `component()` on screens.
3. **`component()`** — builders with `sectionHeader` only.
4. **`componentContent()`** — headerless blocks from `blocks.mjs`.
5. **Spacing** — only via constants in the Agent contract table.
