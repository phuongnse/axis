# Wireframe kit — architecture

> **Navigation**: [Wireframe playbook](../playbooks/wireframes.md) · [← docs/README.md](../README.md)

## Goal

**Reuse blocks** so every screen shares the same geometry, colors, and spacing — not copy-paste in `generate-screens.mjs`.

## What lives in this folder

| File | Role |
|------|------|
| `components.mjs` | Primitives: colors, shapes, `fieldLabel`, `component()` / `componentContent()`, `appShell` |
| `blocks.mjs` | **Auth & form blocks** (SSO row, fields, terms, `authCard`, register-org painters) — **edit here first** for auth UI |
| `generate-template.mjs` | Kit section builders **S01–S37** (table, modal, workflow canvas, …) — import-only, no CLI output |
| `generate-screens.mjs` | Screen generators → `docs/use-cases/{domain}/{use-case}/*.excalidraw` |
| `app-shell.excalidraw` + `.svg` | Shared app chrome (only committed artifact besides source `.mjs` here) |

There is **no** `_template.excalidraw` catalog. Kit geometry is defined in `.mjs` source; visual outputs live under each use-case folder.

## Layers (single source per layer)

| Layer | File | Screens use |
|-------|------|-------------|
| Primitives | `components.mjs` | `rect`, `text`, `fieldLabel`, `component()` |
| Blocks | `blocks.mjs` | `placeAuthExternalSignIn`, `authFormField`, `paintRegisterOrg*`, … |
| Kit sections | `generate-template.mjs` | `component(buildTable, cx, cy)` etc. |

## Workflow

```bash
# 1. Edit source
#    components.mjs   — colors, fieldLabel, REQUIRED_MARKER_GAP
#    blocks.mjs       — auth SSO, fields, register-org painters
#    generate-template.mjs — only when adding/changing S01–S37 builders

# 2. Regenerate screens (twice; second run must produce empty git diff)
node docs/wireframes/generate-screens.mjs
node docs/wireframes/generate-screens.mjs

# Optional: one use-case only
SCREEN_FILTER=register-org node docs/wireframes/generate-screens.mjs

# 3. SVG previews (Kroki)
docs/scripts/generate-wireframes.ps1 -Filter register-org
```

`app-shell` is generated by `generate-screens.mjs` like any other screen; regen all screens or rely on stable seeds when you only touch one use case.

## Rules

1. **New auth / register UI** → `blocks.mjs` (not `generate-template.mjs`).
2. **New large pattern** (data grid, builder layout) → `buildXxx(y0)` in `generate-template.mjs` + `component()` on screens.
3. **`component()`** — builders with `sectionHeader` only.
4. **`componentContent()`** — headerless blocks from `blocks.mjs`; never `component()` on those.
5. **Do not** hand-edit generated `.excalidraw` under use-cases.

## Block inventory (`blocks.mjs`)

| Block | Use on |
|-------|--------|
| `placeAuthExternalSignIn` | register-org, any auth screen with SSO |
| `buildAuthCardHeader` / `buildAuthCardFooter` / `buildAuthSubmitButton` | Composite auth cards |
| `buildAuthCardBrandBar` | Informational cards (email-confirmation, provider errors) |
| `fieldLabelBlock` (`components.mjs`) | Label + optional `?` + help text line |
| `authFormField`, `authReadOnlyValueField`, `authSlugPreviewField`, `authTermsRow` | register-org, states (`helpText` param) |
| `paintRegisterOrgEntryFields`, `paintRegisterOrgCompleteFields` | register-org + `*-states` |
| `authCard` | login, register, forgot/change password, accept invite |

Constants (`AUTH_CARD_W`, `AUTH_EXTERNAL_SIGN_IN_BLOCK_H`, …): import from `blocks.mjs`; do not hardcode `440` on screens.
